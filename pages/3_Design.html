# UX/UI Ключевой вопрос: СКОЛЬКО У ЗАКАЗЧИКА ВРЕМЕНИ И ДЕНЕГ. - В сайтах сравниваем контент, в мобильных приложениях - функции. - Эмпатия - способность собрать информацию о мыслях и чувствах пользователя. Т.е. поставить себя на место пользователя, понять его потребности, страхи и боли. На основе этих знаний спрогнозировать его действия. - Физиология - область фокусировки зрения, параметры рук/ладоней/пальцев - Психология - эмоции, мотивации - Нейробиология - обработка информации, запоминание - Физические ограничения среды - параметры устройства, хранение данных ## Задача почти всегда это - Презентация услуг/продуктов - Повышение лояльности аудитории - Прямая выгода - Конкуренция с текущим методом достижения потребности - Экономия ресурсов (время/энергия/деньги) ## Процесс ### 1. Исследуем (собираем данные) - #### Предварительный сбор данных Поиск и анализ релевантных статей и информации - #### Конкурентный анализ (анализ рынка) - Сбор данных по аналогичным и релевантным решениям - Дешевый и эффективный метод - Использовать всегда - #### Визуальный анализ - Референсы и мудборды - Выявление визуальных закономерностей - #### Анализ ассоциаций и эмоций - Ожидание пользователей - Создание нового эмоционального опыта - Формирование мотивации ### 2. Анализируем (выявляем закономерности) #### Пользовательский проект Помогает сегментировать пользователей и начать их "чувствовать" #### Пользовательский сценарий (набор шагов) на примере рекрутингового приложения - Старт - пользователь открыл приложение - Обучение (онбординг) - пользователь проходит минимальное обучение пользования приложением - Регистрация - пользователь проходит быструю регистрацию (ввод почты, телефона, ФИО) - Заполнение профиля - пользователь заполняет минимально необходимую информацию в профиле - Проект - пользователь создаёт задачу проекта для поиска кандидатов - Публикация - пользователь публикует проект (подтверждает публикацию) - Просмотр откликов - пользователь просматривает отклики на проект - Утверждение кандидатов - пользователь утверждает кандидатов для проекта - Закрытие проекта - пользователь закрывает проект - Финиш - пользователь закрывает приложение ### 3. Делаем выводы (гипотезы) #### Job (User) story Понимание потребности наших пользователей #### Customer (User) Journey Map Расширенная версия пользовательского сценария #### Структура интерфейса (Wireframe) и прототип Предварительный черновик логики интерфейса #### Jobs To Be Done Концентрируемся на функции/услуге нашего интерфейса ### 4. Запускаем продукт (тестируем гипотезы) #### Интервью с пользователями - Перед запуском продукта - Сбор мнений после запуска - Дорогой метод - Нужно "чувствовать" людей ### 5. Проверяем результаты #### Сбор аналитики показателей Обязательный метод [Бесплатный онлайн интенсив UX/UI](https://intensive.skillbox.ru/ui-ux/) [Бесплатный интенсив по User Experience](https://rb.gy/x5gxsy) [База Figma бесплатный курс](https://www.uprock.ru/courses-2/figma) [UX/UI](https://www.youtube.com/watch?v=jpXEDEVXzoo&list=WL&index=3) [Видео-уроки по дизайну:](https://t.me/s/uprockdesign) [Личный инстаграм Евгения: ](https://www.instagram.com/zhenya.ninja/)  [Крутые анимации на dribbble:](https://dribbble.com/uprockpro) [Лучшие сайты на Behance:](https://www.behance.net/uprockawards) # Figma ## База фигмы: - слои - группы - фреймы - сэкшены, - констрейнсы - автолейауты - компоненты - варианты - пропертис - анимация ## Проектирование (UX) - User story - ULM - карта путей пользователя - Прототипирование - черновой вариант интерфейсов. Визуализируем и тестируем как будет работать продукт. - Usability-testing - тестируем на реальных пользователях ## Дизайн-система (UI) - Визуальное решение - референсы - скриншоты понравившихся дизайнов. Копируем цвета, шрифты и т.д. - Токены - переменные с основными параметрами для всех элементов интерфейса: - цвет [ColorScheme.Ru — Цветовой круг он-лайн: Подбор цветов и генерация цветовых схем](https://colorscheme.ru/) Figma plagin - color compass - распарсить оттенки. Создать стили заливки по оттенкам (Fill выбрать заливку, +, Blue/300 = Цвет/Оттенок ). - шрифт - Из прототипа скопировать Заголовки, подзаголовки, основной текст, текст кнопок, ссылок и инпутов. - Сгруппировать текста по шрифту, размеру и толщине. - Подписать каждую группу. - Создать стили текста: выбрать стиль текста, +, название стиля. - отступ - тень - форма - и т.д. - UI-kit - создание компонентов для многократного использования - Копируем составные блоки, которые повторяются (header, footer и т.д.) - Из составных блоков копируем отдельные простые блоки (кнопки, инпуты, изображение, ссылка и т.д.) ## Итоговый дизайн [Figma](https://www.figma.com/) [Templates](https://www.figma.com/@wordpressorg) [Tutorial base figma](https://www.youtube.com/playlist?list=PLgP78sMGTxfBnHmM4UE1KM8R7MdzFdLox) [Tutorial design](https://www.youtube.com/playlist?list=PLs1PNT-L34zrxi6NbbRsRWG0mtDyh-mV6) [Animation](https://www.youtube.com/watch?v=SW2EizGLkMY&t=1091s) [Behance case](https://harvest-virgo-f1b.notion.site/Behance-517e810da89d4aa78066dac7cc92e243) # Resources Все самые полезные сайты и мокапы для дизайнеров: [https://www.baza.uprock.ru/](https://www.youtube.com/redirect?event=video_description&redir_token=QUFFLUhqa0tlVzZzQkliUTZxZzIyNk9Kc0R5ZXZYRDRPUXxBQ3Jtc0trazRLR3MyUW5pRjlLOTVBVlBzODYwR0lldUJQb1pxbVpfaWtJRUNnWmxrQldSTVpjdERJRjNvZWExZ3dOYjdwZ3lzc3BVelhsdnVmUV9EcURpVWd1M0VONlpWdUNJMHFDYndsZ1lLeGJ2Q096Ukwzaw&q=https%3A%2F%2Fwww.baza.uprock.ru%2F&v=jpXEDEVXzoo) ## AI [ChatGPT](https://chatgpt.com/) ### Image [Leonardo](https://app.leonardo.ai/image-generation) ultrarealism, 8k, cinematic [Recraft](https://www.recraft.ai/) ## See [Darkmode](https://www.darkmodedesign.com/) - Сайт, посвященный тренду темного режима в дизайне.
