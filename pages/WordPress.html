Это необходимо для того, чтобы WordPress считал это блочной темой. - README.txt (обзор темы) - screenshot.png (скриншот темы 1200×900) - theme.json(глобальные настройки и стили) - parts(Template Parts): содержит пользовательские части шаблона темы. - patterns(Шаблоны блоков): Многоразовые шаблоны, состоящие из одного или нескольких блоков, которые пользователи могут вставлять через интерфейс редактора. WordPress автоматически регистрирует файлы, включенные в эту папку. - styles(Вариации стилей): вариации глобальных настроек и стилей темы, хранящиеся в отдельных файлах JSON. - templates(Шаблоны): Файлы, которые представляют общую структуру документа front-end. - inc(Пользовательская функциональность): темы часто будут иметь пользовательские PHP-классы или файлы. -.editorconfig( EditorConfig ): используется для настройки форматирования, такого как окончания строк и интервалы, для редакторов кода. - .gitattributes( Git: Атрибуты ): Настраивает атрибуты с помощью системы контроля версий Git. - .gitignore( Git: Ignore ): определяет файлы, которые следует игнорировать при фиксации кода в репозитории Git. - CHANGELOG.md( Ведите журнал изменений ): понятный человеку журнал важных изменений для каждого выпуска вашей темы. - LICENSE.md( Обзор темы: Лицензирование и авторские права ): Определяет лицензию для темы. Обратите внимание, что все темы, отправленные в каталог тем WordPress, должны быть лицензированы по GPL v2+. - package.json( npm: package.json ): часто используется для определения процесса сборки и зависимостей разработки в среде Node. WordPress автоматически обрабатывает окончательную разметку, поэтому вам остается только создать шаблоны.
